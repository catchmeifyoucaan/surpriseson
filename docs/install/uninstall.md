# Uninstall

Two paths:
- **Easy path** if `surprisebot` is still installed.
- **Manual service removal** if the CLI is gone but the service is still running.

## Easy path (CLI still installed)

Recommended: use the built-in uninstaller:

```bash
surprisebot uninstall
```

Non-interactive (automation / npx):

```bash
surprisebot uninstall --all --yes --non-interactive
npx -y surprisebot uninstall --all --yes --non-interactive
```

Manual steps (same result):

1) Stop the gateway service:

```bash
surprisebot daemon stop
```

2) Uninstall the gateway service (launchd/systemd/schtasks):

```bash
surprisebot daemon uninstall
```

3) Delete state + config:

```bash
rm -rf "${SURPRISEBOT_STATE_DIR:-$HOME/.surprisebot}"
```

If you set `SURPRISEBOT_CONFIG_PATH` to a custom location outside the state dir, delete that file too.

4) Delete your workspace (optional, removes agent files):

```bash
rm -rf ~/surprisebot
```

5) Remove the CLI install (pick the one you used):

```bash
npm rm -g surprisebot
pnpm remove -g surprisebot
bun remove -g surprisebot
```

6) If you installed the macOS app:

```bash
rm -rf /Applications/Surprisebot.app
```

Notes:
- If you used profiles (`--profile` / `SURPRISEBOT_PROFILE`), repeat step 3 for each state dir (defaults are `~/.surprisebot-<profile>`).
- In remote mode, the state dir lives on the **gateway host**, so run steps 1-4 there too.

## Manual service removal (CLI not installed)

Use this if the gateway service keeps running but `surprisebot` is missing.

### macOS (launchd)

Default label is `com.surprisebot.gateway` (or `com.surprisebot.<profile>`):

```bash
launchctl bootout gui/$UID/com.surprisebot.gateway
rm -f ~/Library/LaunchAgents/com.surprisebot.gateway.plist
```

If you used a profile, replace the label and plist name with `com.surprisebot.<profile>`.

### Linux (systemd user unit)

Default unit name is `surprisebot-gateway.service` (or `surprisebot-gateway-<profile>.service`):

```bash
systemctl --user disable --now surprisebot-gateway.service
rm -f ~/.config/systemd/user/surprisebot-gateway.service
systemctl --user daemon-reload
```

### Windows (Scheduled Task)

Default task name is `Surprisebot Gateway` (or `Surprisebot Gateway (<profile>)`).
The task script lives under your state dir.

```powershell
schtasks /Delete /F /TN "Surprisebot Gateway"
Remove-Item -Force "$env:USERPROFILE\.surprisebot\gateway.cmd"
```

If you used a profile, delete the matching task name and `~\.surprisebot-<profile>\gateway.cmd`.

## Normal install vs source checkout

### Normal install (install.sh / npm / pnpm / bun)

If you used `https://surprisebot.bot/install.sh` or `install.ps1`, the CLI was installed with `npm install -g surprisebot@latest`.
Remove it with `npm rm -g surprisebot` (or `pnpm remove -g` / `bun remove -g` if you installed that way).

### Source checkout (git clone)

If you run from a repo checkout (`git clone` + `pnpm surprisebot ...` / `bun run surprisebot ...`):

1) Uninstall the gateway service **before** deleting the repo (use the easy path above or manual service removal).
2) Delete the repo directory.
3) Remove state + workspace as shown above.
